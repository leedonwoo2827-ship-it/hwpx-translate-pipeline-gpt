# translate-ko — 영문 학술 원문 → 한국어 완역 + 내부용 분석

파이프라인 **02-translate** 단계. 입력 = 영문 원문(01-extract의 `content.en.md`).
출력 = 한국어 완역 + 장 말미 분석 섹션(02-translate의 `content.ko.md`).
지금은 Claude가, 향후 로컬 LLM이 동일 계약으로 수행. (자연스러움 다듬기는 다음 단계
[refine-ko](../refine-ko/SKILL.md)에서.)

## 번역 원칙

1. **완역·충실성**: 문단·문장을 빠짐없이 옮긴다. 사실·수치·인용·논지를 보존하고 임의 요약·
   삭제·의역 금지. 원문에 없는 정보 추가 금지.
2. **문체**: 한국어 학술 문어체(‘~이다/~한다’ 평서형). 전문 용어는 표준 역어 + 첫 등장 시
   원어 병기(예: 전이(transfer), 인식론적 정당화(epistemic legitimation)).
3. **용어 일관성**: 책 전체에서 핵심 용어를 일관되게(전이/전문성/개념적·인식론적 지식/
   관계적 대학/행성적 주체성/톨릭 아담 등). 인명은 한글 음역 + 원어 병기.
4. **가독성**: 직역투를 피하고 자연스러운 한국어로. (다만 최종 윤문은 refine-ko가 담당.)

## 포맷 규칙 (문서공방 hwpx 마크다운 계약)

- `# 제목1`(장 제목), 그 아래 `**저자**` 한 줄. `## 제목2`(소제목). 헤딩에 마크다운 기호 금지.
- 문단=본문, `- `=○ 1단계 불릿, `  - `=− 2단계, `> `=인용(에피그래프 포함), `**강조**`.
- 도표: 원문 `![...](figures/fig-0N.png)` 링크를 **경로·파일명 그대로** 유지(캡션만 한국어로).
  중복 캡션(`> Figure N:`)은 제거하고 이미지 alt(캡션)만 남긴다.
- 참고문헌은 “원문 참고문헌 목록 참조(저자 연도; …)”로 축약 가능.

## 분석 섹션 (내부용, 편집팀 추가)

장 말미에 다음을 덧붙인다. 원문에 없음을 인용문으로 명시:
```
## 분석 (내부 검토용)

> 이 절은 원문에 없으며, 내부 연구·활용을 돕기 위해 편집팀이 덧붙인 요약·논점·시사점이다.

### 요약
### 핵심 논점
### 국내 대학·정책에의 시사점
```
- 요약: 장의 핵심 명제·구조를 5~8문장.
- 핵심 논점: 굵게 표제 + 한 줄 설명의 불릿 5개 내외.
- 국내 시사점: 한국 대학·정책 맥락으로의 함의 불릿 5개 내외.

## 절차 (한 장)

1. `content.en.md`를 통독하고 구조(제목/소제목/도표/참고문헌) 파악.
2. 위 원칙·포맷으로 완역 → 도표 링크 유지 → 분석 섹션 작성.
3. 02-translate의 `content.ko.md`로 저장. 이후 refine-ko로 윤문.
